# `/pay` Payment-Details Page — Design

**Date:** 2026-07-15
**Status:** Approved, ready for implementation plan

## Goal

Publish a standalone page at `www.swiss-controls.com/pay` that shows Rigitrade AG's
bank payment details to clients who need to pay an invoice. The details must be:

- **Not indexed** by search engines (bank details should not surface in search).
- **Not copyable / not machine-readable** from the page source (rendered as a raster
  image, not selectable text).
- Reachable from the site footer via a **Swiss-flag icon**.

Host is **Vercel** (static export, `output: "export"`).

## Route & placement

The site uses `next-intl` with `localePrefix: "always"`, so all real pages live under
`/en/`. The root `/` is a pass-through redirect shell. The `/pay` URL must have **no
locale prefix**, so it lives **outside** the `[locale]` tree, mirroring the root page:

- `src/app/pay/layout.tsx` — supplies its own `<html>`/`<body>` (root layout is a
  pass-through that returns `children`), applies the font-variable classes and
  `bg-paper text-ink` like the locale layout, imports nothing extra (globals.css is
  already imported by the root layout). Exports `metadata` with
  `robots: { index: false, follow: false }`.
- `src/app/pay/page.tsx` — the branded content.

With `trailingSlash: true`, the built URL is `/pay/` (`out/pay/index.html`). Vercel
redirects `/pay` → `/pay/`.

## Page content (branded, no site nav)

Centered column on the paper background:

1. Swiss Controls **wordmark** (existing `<Wordmark>` component).
2. Heading: **"Payment Details"** in site typography.
3. The **details image** (see below).
4. One line of real (non-sensitive) text:
   *"Please include your invoice number as the payment reference."*

No site header, no footer nav — this is a focused invoice-companion page reached only
via the direct link or the footer flag.

## The details image (non-copyable)

- Committed raster: `public/pay/bank-details.png`, served at `/pay/bank-details.png`.
- Generated by a reproducible script `scripts/generate-pay-image.ts`, run via a new
  npm script `gen:pay-image`. The script:
  - Uses **Playwright** (already a dev dependency) headless Chromium.
  - Renders a styled HTML card containing the payment details (site-consistent clean
    sans, e.g. `system-ui`/Segoe UI stack for offline reproducibility).
  - Screenshots the card element at `deviceScaleFactor: 3` for crispness.
  - Writes the PNG to `public/pay/bank-details.png`.
  - Payment-detail values live in a single object at the top of the script so a future
    change is a one-line edit + re-run.
- `<img>` `alt` is a **generic** label — "Swiss Controls bank payment details, shown as
  an image" — deliberately **not** the IBAN/account numbers, so no sensitive text sits
  in the HTML source (which would be copyable and scrapeable, defeating the purpose).

**Accessibility tradeoff (accepted):** the numbers are not available to screen readers
or copy-paste. This is the explicit client requirement.

## Footer Swiss-flag link

In `src/components/interactive/site-footer-grid.tsx`, in the bottom-bar icon cluster
(currently Mail + WhatsApp), add a **Swiss flag icon**:

- Inline two-color SVG: red rounded square background + white cross (cannot use the
  `currentColor → hover:text-red` trick the mono icons use, because the flag is
  multicolor).
- Plain `<a href="/pay/">` (NOT the next-intl `Link`, since `/pay` is outside the
  locale tree). `aria-label="Payment details"`, `target` stays same-tab.
- Hover affordance: subtle opacity and/or scale transition, matching the existing
  icons' `h-5 w-5` sizing.

## Keeping it out of search (belt-and-suspenders)

1. **Meta tag (guaranteed):** page `metadata.robots = { index: false, follow: false }`
   → emits `<meta name="robots" content="noindex, nofollow">` in the static HTML.
2. **Sitemap + robots.txt:** in `next-sitemap.config.js`, add `/pay` to `exclude` and a
   `Disallow: /pay` entry to the `robotsTxtOptions` policy.
3. **HTTP header (Vercel):** add a `vercel.json` with an `X-Robots-Tag: noindex,
   nofollow` header scoped to `/pay` (and `/pay/bank-details.png`), so even the raw
   image response is marked noindex.

## Out of scope (YAGNI)

- "Download PDF" button / serving an actual PDF file.
- i18n / localized version of this page.
- Any structured or text representation of the numbers.
- Linking `/pay` from anywhere other than the footer flag icon.
